## [Unreleased]

## [0.1.0] - 2026-05-24

### Added

- Added best-effort startup/reload changelog notifications for this Pi package. The extension records the last seen `package.json` version under Pi's agent directory, skips fresh installs, and displays newly versioned `CHANGELOG.md` sections once after `pi update` installs a newer package version.
- Added a manifest/changelog regression test requiring the current `package.json` version to have a matching `## [x.y.z]` changelog section, so release PRs cannot leave user-visible notes only under `Unreleased`.
- Added changelog-entry signatures to the extension startup state so same-version Git package updates and legacy state created by the first changelog implementation can still display the current versioned changelog once.

### Changed

- Audited public docs against the current extension/source state: refreshed repository/index dates, updated extension and status docs to mention all three slash commands, restored the missing thinking-block replay section in `docs/current-status.md`, updated `REPO_MAP.md`'s docs tree for the moved rationale/model/slash-command pages, and corrected the `docs/rationale.md` customization note so it no longer implies Pi `modelOverrides` affect extension-registered provider metadata.
- Tightened the `@types/node` constraint from `^22.19.0` to `~22.19.0` so typings stay on the exact Node floor minor while still allowing Node `22.19.x` patch updates. Added a Dependabot ignore rule for `@types/node` minor and major version updates so automated updates cannot widen the typings beyond the supported runtime floor; patch updates remain allowed.
- Pinned `@types/node` from `^25.9.0` (Node 25 typings) to `^22.19.0` so the devDependency tracks the `engines.node` floor (`>=22.19.0`). Previously TypeScript could resolve Node 25-only APIs at typecheck time even though the package declares Node 22.19 as its minimum runtime, allowing accidental use of types not present on the supported runtime. Lockfile now resolves `@types/node` to `22.19.19`. Note: `@types/node`'s `dist-tags` map `ts6.0` to `25.9.0`, but our TypeScript 6.0 typecheck (`tsc --noEmit`) passes against `22.19.x` typings; the dist-tag is a recommendation, not a constraint. Verified by `npm run check` (143 pass / 1 skipped / 0 fail, tsc clean).
- Tightened the README intro (3 paragraphs trimmed of redundant qualifiers without changing meaning) and clarified terminology to match Pi's own [packages.md](https://github.com/earendil-works/pi/blob/main/docs/packages.md) taxonomy: the distribution unit is a *Pi package*, the code module inside it is an *extension*. Updated `package.json` `description` from "Pi extension for using Claude Code subscription OAuth through a native provider" to "Pi package providing a native Claude subscription provider that authenticates with Claude Code OAuth", and removed the non-canonical `pi-extension` keyword (Pi's gallery taxonomy only documents `pi-package`; `pi-package` is retained). Updated `AGENTS.md` first line to match.
- README `Verify the extension registered` and `docs/slash-commands.md` troubleshooting steps now point at `pi list` (the actual CLI command, per `pi --help`) instead of the non-existent `pi extensions list`.
- Updated registered model output caps to match Pi 0.75.5's latest Anthropic metadata while preserving subscription-context limits: Haiku 4.5 and Sonnet 4.6 now cap output at 64,000 tokens; Opus 4.6 and Opus 4.7 now cap output at 128,000 tokens; Sonnet 4.6 remains at a 200,000-token context window because the Claude Code subscription path targeted by this package provides 200,000 tokens. Opus 4.7 now carries local `compat.forceAdaptiveThinking` metadata, and the native stream path honors that flag; Sonnet/Opus 4.6 remain on manual `budget_tokens` by design.
- Renamed the README `Model reference` table column `Default max_tokens` to `Output cap` and added a paragraph documenting that the actual `max_tokens` sent to Anthropic is `min(requestedOutputTokens + thinkingBudget, output_cap)` after the manual-thinking-budget fix (PR #24). Linked to a new `docs/current-status.md` § Manual thinking budgets section that documents the composition, the per-model output caps, the Pi-level-to-budget mapping, the budget-reduction/budget-drop fallback when the cap forces an invalid payload, and the test coverage.
- `docs/current-status.md` `Updated:` stamp moved from `2026-05-07` to `2026-05-18`.

### Fixed

- Documented Pi subagent usage with this provider: subagent prompts appended with `--append-system-prompt` stay intact behind the required separate Claude Code identity block, but bundled example agents that specify `claude-sonnet-4-5` must be edited to a supported provider-qualified model such as `claude-subscription/claude-sonnet-4-6`. The extension continues not to register unverified model ids and blocks known non-subscription Anthropic routing to avoid accidental API-key or extra-usage billing.
- Native Claude subscription response-start watchdogs now default to 120s while keeping the post-response SSE body no-progress watchdog at 45s. Anthropic cannot send streaming `ping` events before response headers exist, so using the in-stream ping budget for response start caused false-positive `made no progress for 45000ms before response started` errors on large cached Opus requests. Explicit `streamNoProgressTimeoutMs` / `timeoutMs` overrides still apply. Covered by a response-start timeout regression test.
- Native replay now drops assistant `toolCall` blocks unless every tool call has a matching immediately-following `toolResult`, and skips orphan tool results. This prevents resumed sessions with interrupted/mismatched tool history from sending invalid Anthropic `tool_use` / `tool_result` sequences that fail with `400 invalid_request_error`. Covered by an incomplete-tool-result replay regression test.
- Native Claude subscription streams now treat streamed tool calls as provisional until a clean Anthropic `message_stop`; if a stream fails first, errored assistant messages drop `toolCall` blocks while preserving non-tool partial content and redacted error details. Stream no-progress handling now covers both stalled response bodies and requests that never receive response headers, surfacing a clear timeout instead of leaving Pi indefinitely on `Working...`. Covered by interrupted-tool-call and no-progress regression tests.
- Manual-budget thinking models (`claude-haiku-4-5`, `claude-sonnet-4-6`, `claude-opus-4-6`) no longer send invalid Anthropic payloads when the caller clamps visible output (notably Pi 0.75 compaction, which routes summary requests through the custom provider with `maxTokens ≈ 8192` while extension thinking budgets reach `20480`/`32768`). Anthropic requires `max_tokens > thinking.budget_tokens`; the previous code wrote the caller's output ask directly into `max_tokens`, producing `budget_tokens >= max_tokens` and a `400 invalid_request_error`. `contextToPayload` in `src/native-stream-simple.ts` now treats `options.maxTokens` as the desired visible-output budget, sets Anthropic `max_tokens = min(requestedOutputTokens + thinkingBudget, model.maxTokens)`, reduces `thinking.budget_tokens` only when the model cap forces an otherwise-invalid payload, and omits the thinking block entirely when no budget at or above Anthropic's `1024`-token minimum fits. Opus 4.7's adaptive thinking path is unchanged. Covered by three new `manual-budget thinking ...` tests in `tests/native-stream-simple.test.ts` plus the existing `models that can avoid adaptive thinking use budget tokens` coverage.
