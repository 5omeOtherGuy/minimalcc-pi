# Changelog

All notable changes to this project are documented here.

## [Unreleased]

### Changed

- Scoped the `interleaved-thinking-2025-05-14` Anthropic beta header to the models that actually need it. It was previously sent on every native Messages request as a base beta, but adaptive-thinking models (Opus 4.7+/Fable 5) imply interleaved thinking server-side; the header is only load-bearing for manual-budget thinking models (Haiku 4.5, Sonnet 4.6, Opus 4.6). `buildNativeMessagesRequest` now derives the signal from the payload (`thinking.type === "enabled"`) and sets a new `interleavedThinking` `NativeHeaderOptions` flag, mirroring the existing payload-driven `serverSideFallback`/`fallbacks` coupling; adaptive thinking and requests with no thinking omit the beta. Headers do not participate in the prompt-cache prefix, so outgoing request bodies are byte-identical and the golden body snapshots are unchanged. Covered by header tests in `tests/native-request.test.ts` and `tests/native-credentials.test.ts`.

- Throttled incremental tool-input partial-argument parsing for large streamed tool calls. The native stream re-parses the full accumulated `input_json_delta` fragment to keep `partial` tool arguments fresh, which is quadratic in the final input size when done on every delta (measured at 128-byte deltas: ~74 ms of CPU for a 16 KiB input, ~24 s for 270 KiB, ~5 min for 1 MiB, all blocking the event loop mid-stream). Inputs up to 16 KiB keep the historical parse-on-every-delta behavior; above that, partial arguments are re-parsed only when the accumulated fragment has grown 25% since the last parse, bounding total incremental parse work to a constant multiple of the input size. `toolcall_delta` events are still emitted for every delta, and the final arguments at `content_block_stop` are still parsed exactly (including the `edit` normalizer), so executed tool calls are unchanged. Covered by a new throttling regression test in `tests/native-stream-simple.test.ts`.

- Made the opt-in microcompaction feature discoverable to end users: `README.md` now lists the `PI_CLAUDE_MICROCOMPACT`, `PI_CLAUDE_MICROCOMPACT_KEEP_RECENT`, and `PI_CLAUDE_MICROCOMPACT_MIN_BYTES` environment variables in the Configuration table, adds the `/claude-subscription-microcompaction` command to the slash-command table (now four commands), and notes the feature under “What it provides”. The `/claude-subscription-status` command now also points at the optional feature and its enable flag in its single info notification. No behavior change; microcompaction remains off by default.
- Post-stream telemetry now records after the assistant `done` event instead of before it. The request-shape fingerprint hashes a deep key-sorted stringify of the whole request body (including full message history) at ~7 ms per MB of body; computing it before `done` delayed Pi's continuation (tool execution, next turn) by that much on every successful request in long sessions. The fingerprint, cache-diagnostics sample, and usage-telemetry record are now computed on a `setImmediate` after the stream ends, in completion order, with failures swallowed as best-effort diagnostics. The recorded values are unchanged; `/claude-subscription-usage` and `/claude-subscription-cache-diagnostics` see the new sample one event-loop turn later. Stream tests drain the deferred work via a shared helper before asserting.

### Added

- Added `claude-fable-5` to the `claude-subscription` provider with a 1,000,000-token context window, 128,000-token output cap, text/image input, and the shifted adaptive-Opus effort mapping. Fable 5's thinking is always on server-side: requests send `thinking: { type: "adaptive", display: "summarized" }` when Pi reasoning is enabled and omit the `thinking` parameter entirely when it is off (an explicit `disabled` is a 400 on Fable 5); sampling parameters are never sent. Covered by `tests/native-fable-5.test.ts` and the model-matrix/golden suites.
- Added server-side refusal fallback for Fable 5 (matching native Claude Code behavior): Fable requests carry `fallbacks: [{ model: "claude-opus-4-8" }]` plus the `server-side-fallback-2026-06-01` beta header, so a safety-classifier decline (`stop_reason: "refusal"`) is retried on Opus 4.8 by the API in the same round trip — the cached prompt prefix is never re-read client-side after the model switch. The SSE parser and stream now understand the `fallback` content-block marker (audit-only, never Pi content) and treat it as the replay boundary: pre-boundary thinking signatures are cleared and closed pre-boundary tool calls are dropped so the echoed turn satisfies Anthropic's fallback replay rules. A terminal refusal (whole chain declined) surfaces a descriptive error including the `stop_details` policy category instead of the generic unsupported-stop-reason failure. If the OAuth lane rejects the `fallbacks` beta, the request retries once without it and fallbacks latch off for the process.

- Native Anthropic Messages requests now keep the pooled TLS connection to `api.anthropic.com` alive for 60 s between requests (new `src/native-fetch-dispatcher.ts`, configurable via `PI_CLAUDE_HTTP_KEEPALIVE_MS`, `0` disables). undici's default 4 s keep-alive plus Anthropic sending no `Keep-Alive` hint meant nearly every agent turn separated by tool runs or typing paid a fresh DNS + TCP + TLS handshake; measured live, that re-handshake added ~250-450 ms to time-to-first-byte versus a reused connection, while Anthropic's edge held idle connections >= 130 s (so a 60 s client-side timeout still always closes first and adds no stale-socket risk over the default). The dispatcher is a per-fetch undici `EnvHttpProxyAgent` (proxy env behavior unchanged, `allowH2: false` matching Pi), pinned to Pi's exact undici version; if the runtime's fetch rejects the dispatcher's handler protocol (`UND_ERR_INVALID_ARG`, raised synchronously before any bytes are sent), the request transparently retries once without it and keep-alive stays disabled for the process. Covered by `tests/native-fetch-dispatcher.test.ts` and new transport tests in `tests/native-stream-simple.test.ts`.
- Added opt-in Claude Code-style keep-recent microcompaction for the native Claude subscription provider. When `PI_CLAUDE_MICROCOMPACT` is truthy, the provider projects the message array before Anthropic conversion (in `src/native-stream-simple.ts`, via the pure `projectMessagesForNativeMicrocompaction` helper in `src/native-microcompaction.ts`): old, large, text-only, non-error tool results that the request would already send are replaced in place with the `[Old tool result content cleared]` placeholder, keeping the most recent `PI_CLAUDE_MICROCOMPACT_KEEP_RECENT` results (default 5, floored to 1) full. Clearing only applies when it frees at least `PI_CLAUDE_MICROCOMPACT_MIN_BYTES` bytes (default 65536, measured with `Buffer.byteLength`); otherwise the original message array is returned unchanged. The Pi session transcript is never mutated. Tool-result eligibility reuses one shared sequencing module (`src/native-tool-sequencing.ts`, also consumed by `convertMessages`) so tool_use/tool_result invariants and signed-thinking replay are preserved. Redacted per-process counters live in their own store (`src/native-microcompaction-telemetry.ts`, not the cache-diagnostics contract) and are surfaced by the new `/claude-subscription-microcompaction [reset]` command. Disabled by default. Covered by `tests/native-tool-sequencing.test.ts`, `tests/native-microcompaction.test.ts`, and new wiring tests in `tests/native-stream-simple.test.ts`. Cache-edit/`cache_reference` microcompaction remains deferred (absent from the shipped Claude Code CLI and unverified on the OAuth route).
- Added a local `reset` argument to the diagnostics slash commands: `/claude-subscription-usage reset` clears the per-process token/cache/request counters (including the persistent aggregate totals that survive sample eviction) and `/claude-subscription-cache-diagnostics reset` clears recorded cache-read-drop events and per-session comparison state, both without restarting Pi and without regenerating the per-process fingerprint salt. Each replies with a single info notification and no counters; any other argument falls through to the normal summary. Covered by new reset tests in `tests/current-provider-system-shape.test.ts` and documented in `docs/slash-commands.md`, `README.md`, and `docs/current-status.md`.
- Added `docs/verification-gates.md` documenting focused test gates by change type plus supply-chain and runtime drift policy (Pi dependency lockstep, Node floor/typings alignment, Claude Code/Anthropic re-extraction, and extraction-artifact hygiene), linked from `docs/INDEX.md`, `CONTRIBUTING.md`, and `docs/current-status.md`. Added `tests/dependency-drift.test.ts` asserting the `@earendil-works/pi-ai` / `@earendil-works/pi-coding-agent` lockstep (including the nested pi-ai copy), the Node `>=22.19.0` floor and `~22.19.0` typings, and that the drift-sensitive gate categories stay documented.
- Added a Threat model section to `SECURITY.md` documenting the credential-exfiltration, billing-route-confusion, outbound-URL/token, tool-output-leakage, malformed-SSE, denied-tool-call, and destructive-automation abuse cases and their current defenses, and published a Model compatibility matrix in `docs/current-status.md`. Added `tests/model-matrix.test.ts` to keep the matrix in sync with the `MODELS` constants (native-id mapping, context windows, output caps, adaptive-vs-manual thinking) and to assert the threat-model headings stay present without embedding secret-shaped values.

The interactive provider remains on streaming `/v1/messages`, so synchronous output caps are unchanged and the beta is not sent by normal Pi sessions.

### Fixed

- Native streamed `edit` tool calls are now normalized into Pi's exact `edit` schema before Pi validates them, fixing recurring Anthropic-only `Validation failed for tool "edit"` aborts. Claude intermittently emits valid JSON that the transport cannot reject but Pi's `additionalProperties: false` edit schema does: a stray annotation key on an edit item alongside valid `oldText`/`newText` (e.g. `newText_unused`, `_x`, `structuredPatch`; `edits.N: must not have additional properties`), or the whole `edits` array serialized as a JSON string (`edits.0: must be object`). A new `src/edit-tool-arguments.ts` normalizer parses a stringified `edits` array and reduces each item to exactly `{oldText, newText}` when both are strings; the stream path applies it only to the built-in `edit` tool, at both the final tool-input parse and the inline tool_use-start payload. It is deliberately conservative: non-`edit` tools, all other top-level keys, malformed items, and non-array edits pass through untouched so genuine errors still surface. Covered by `tests/edit-tool-arguments.test.ts` and new end-to-end stream cases in `tests/native-stream-simple.test.ts`.
- Native Claude subscription stream-stall errors now include additional safe request/progress diagnostics — model/response ids, `max_tokens`, thinking mode/effort, tool count, serial tool-choice flag, open content-block count, active tool name, accumulated tool-input byte/delta counts, start-input key count, and cache/output usage counters — without logging raw tool arguments, key names, paths, command strings, prompts, or credentials. Covered by a deterministic body-stall-mid-`toolUseInputDelta` regression test.
- Native Claude subscription SSE streams no longer abort active long tool-input generations at Pi's forwarded `timeoutMs` wall-clock boundary (300s by default). Incremental streaming now uses `timeoutMs` only for response-start transport aborts and relies on the body no-progress watchdog after headers arrive, so large `write`/`edit` tool JSON streams can continue while still making progress instead of failing with `Request timed out after 300000ms [... last_event=toolUseInputDelta ...]`. Covered by a deterministic Opus 4.8 large-tool-input regression test that also verifies the default `max_tokens: 128000` amplifier.
- Native assistant replay now sends signed `thinking` and `redacted_thinking` blocks from the same Claude subscription model byte-for-byte: the provider no longer surrogate-sanitizes or trim-filters signed thinking text, and no longer drops signed-but-empty thinking blocks. Anthropic computes the thinking signature over the exact original characters and rejects any change to a thinking block in the latest assistant message with a 400 ("`thinking` or `redacted_thinking` blocks in the latest assistant message cannot be modified"); mutating or removing a block during a tool-continuation turn could trip that error. Cross-model/cross-provider reasoning is still converted to plain assistant text (and sanitized) and never replayed with a signature. Covered by new same-model replay regression tests in `tests/native-stream-simple.test.ts`.
- Native tool requests now use the standard Anthropic Messages API tool schema shape (`name`, `description`, `input_schema`, plus optional `cache_control`) without the non-standard `eager_input_streaming` field or the legacy fine-grained tool-streaming beta. This avoids asking Opus 4.8 for the fragile streamed tool-input mode that produced malformed final tool JSON in live sessions. Covered by request-shape regression tests.
- Native streamed tool input now fails closed when the final `input_json_delta` payload is non-empty but unparseable or not a JSON object, instead of collapsing it to `{}` and surfacing a misleading tool validation error. Error diagnostics report safe metadata such as payload length without logging tool arguments. Covered by parser and stream regression tests.
- `streamNativeMessagesSseEvents` now invokes the `onResponse` hook inside the request's cleanup `try`, so a hook that throws before the SSE body is consumed releases the no-progress timeout and abort listener instead of leaking them. Covered by a new regression test in `tests/native-stream-simple.test.ts`.

### Changed

- Native streamed tool-input parsing now tries raw JSON before lazily computing repaired/completed fallback candidates, instead of eagerly scanning/building every candidate for each `input_json_delta`. This preserves the same parse/repair order while reducing avoidable work for already-valid tool JSON fragments.

- Native streaming now uses a single body no-progress watchdog per response instead of allocating and clearing one timeout per SSE chunk, and scans SSE frames without building a full `split()` line array per frame. This preserves the same stalled-stream failure behavior while reducing timer/allocation churn on long active streams, especially large streamed tool-input JSON. Covered by focused timer-churn/backpressure regression tests, the existing no-progress timeout tests in `tests/native-stream-simple.test.ts`, and the SSE parser suite.
- Credential loading no longer re-reads the credential file (or re-execs `/usr/bin/security` against the macOS Keychain) on every request. A loaded access token is cached in memory keyed by credential path, but only when it carries a finite numeric `expiresAt`, and it is served only while it clears the same 5-minute near-expiry margin the refresh logic uses - so a cached token is never staler than what a fresh file read would have accepted. The 401 force-refresh retry path invalidates the cache first and re-reads from disk/Keychain, so token revocation and account switches heal exactly as before; rotated-on-disk tokens are picked up at the latest when the cached token nears expiry. The native stream also now starts credential loading and Pi-to-Anthropic payload conversion concurrently instead of serially. Covered by new cache tests in `tests/native-credentials.test.ts` (including a Keychain exec-once test) and an overlap-ordering test in `tests/native-stream-simple.test.ts`.

- Native Anthropic tool requests now omit `tool_choice` entirely (so Anthropic applies its `auto` default and parallel tool calls are allowed), for tool-call concurrency parity with Pi's built-in Anthropic provider and Claude Code. This replaces the earlier forced `tool_choice: { type: "auto", disable_parallel_tool_use: true }`: that wire flag was a provider extension's only lever over Pi's harness-owned tool execution, but it is unnecessary because Pi runs a message's tool calls in parallel by default and its built-in `edit`/`write` tools already serialize same-file mutations through a per-realpath mutation queue that applies regardless of provider. Residual exposure (concurrent `bash`, or `bash` + same-file `edit`/`write`) matches the default Pi and Claude Code accept. Note: we can revisit (e.g. re-add the serial flag) if real parallel-tool-call races appear. Covered by updated request-shape and stall-diagnostics regression tests in `tests/native-request.test.ts` and `tests/native-stream-simple.test.ts`.

- Native streaming now defers the per-request cache-diagnostics fingerprint (deep key-sorted stringify + SHA-256 over the entire request body, including full message history) until after the response stream completes, instead of computing it synchronously before `fetch`. The hash is pure diagnostics and is never read mid-stream, so moving it off the pre-send critical path removes that CPU work from time-to-first-token (the effect grows with conversation size) while the recorded `requestFingerprint` and cache-read-drop `changedSections` stay byte-identical. The fingerprint is taken from the actual streamed body, including the rebuilt body on a 401 auth retry. Verified by the existing telemetry/cache-diagnostics stream tests in `tests/native-stream-simple.test.ts`.
- Added an explicit npm package `files` allowlist so published/installable package contents are limited to runtime source, extension entry, docs, README, changelog, and license files instead of inheriting `.gitignore` packaging defaults that included tests, GitHub automation, agent instructions, and local audit reports. Package-manifest tests now assert manifest metadata and deterministic `npm pack --dry-run --json` contents.
- Advanced the dev/test toolchain in lockstep to Pi `0.78.1`: both `@earendil-works/pi-ai` and `@earendil-works/pi-coding-agent` devDependencies (and the lockfile) now pin `^0.78.1`, and `tsx` moves to `^4.22.4`. Both packages must move together: bumping `@earendil-works/pi-ai` alone (as the Dependabot group PR did) leaves a different `pi-ai` nested under `@earendil-works/pi-coding-agent`, and the two copies' `AssistantMessageEventStream` types have separate private members, which fails `tsc` with `TS2322` at the provider registration. `peerDependencies` stay `*` so the package still loads under older/newer Pi at runtime. Verified by a clean `npm ci` + `npm run check` (172 passing tests, typecheck clean). Supersedes the Dependabot PR (#50).
- Startup/reload package changelog notifications now use Pi 0.78.1's optional `ctx.mode` when available, so they remain visible in the TUI but no longer emit unsolicited RPC notifications. Older Pi runtimes that do not expose `ctx.mode` keep the existing `ctx.hasUI` behavior. Covered by a mode-aware startup notification regression test.
- Extracted the self-contained partial `tool_use` JSON argument repair/parsing logic out of the oversized `src/native-stream-simple.ts` into a focused, pure `src/tool-json-arguments.ts` module and added direct deterministic unit tests in `tests/tool-json-arguments.test.ts`. ### Fixed

- Fixed startup/reload changelog option resolution from the nested extension entry path so it reads the package root `package.json` and `CHANGELOG.md` instead of the `extensions/` directory.
- Corrected cache-diagnostics documentation to describe the implemented per-process salted SHA-256 fingerprints rather than HMAC.

- Documented Pi subagent usage with this provider: subagent prompts appended with `--append-system-prompt` stay intact behind the required separate Claude Code identity block, but bundled example agents that specify `claude-sonnet-4-5` must be edited to a supported provider-qualified model such as `claude-subscription/claude-sonnet-4-6`. The extension continues not to register unverified model ids and blocks known non-subscription Anthropic routing to avoid accidental API-key or extra-usage billing.
- Native Claude subscription response-start watchdogs now default to 120s while keeping the post-response SSE body no-progress watchdog at 45s. Anthropic cannot send streaming `ping` events before response headers exist, so using the in-stream ping budget for response start caused false-positive `made no progress for 45000ms before response started` errors on large cached Opus requests. Explicit `streamNoProgressTimeoutMs` / `timeoutMs` overrides still apply. Covered by a response-start timeout regression test.
- Native replay now drops assistant `toolCall` blocks unless every tool call has a matching immediately-following `toolResult`, and skips orphan tool results. This prevents resumed sessions with interrupted/mismatched tool history from sending invalid Anthropic `tool_use` / `tool_result` sequences that fail with `400 invalid_request_error`. Covered by an incomplete-tool-result replay regression test.
- Native Claude subscription streams now treat streamed tool calls as provisional until a clean Anthropic `message_stop`; if a stream fails first, errored assistant messages drop `toolCall` blocks while preserving non-tool partial content and redacted error details. Stream no-progress handling now covers both stalled response bodies and requests that never receive response headers, surfacing a clear timeout instead of leaving Pi indefinitely on `Working...`. Covered by interrupted-tool-call and no-progress regression tests.
- Manual-budget thinking models (`claude-haiku-4-5`, `claude-sonnet-4-6`, `claude-opus-4-6`) no longer send invalid Anthropic payloads when the caller clamps visible output (notably Pi 0.75 compaction, which routes summary requests through the custom provider with `maxTokens ≈ 8192` while extension thinking budgets reach `20480`/`32768`). Anthropic requires `max_tokens > thinking.budget_tokens`; the previous code wrote the caller's output ask directly into `max_tokens`, producing `budget_tokens >= max_tokens` and a `400 invalid_request_error`. `contextToPayload` in `src/native-stream-simple.ts` now treats `options.maxTokens` as the desired visible-output budget, sets Anthropic `max_tokens = min(requestedOutputTokens + thinkingBudget, model.maxTokens)`, reduces `thinking.budget_tokens` only when the model cap forces an otherwise-invalid payload, and omits the thinking block entirely when no budget at or above Anthropic's `1024`-token minimum fits. Opus 4.7's adaptive thinking path is unchanged. Covered by three new `manual-budget thinking ...` tests in `tests/native-stream-simple.test.ts` plus the existing `models that can avoid adaptive thinking use budget tokens` coverage.
